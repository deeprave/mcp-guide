## ADDED Requirements

### Requirement: Onboarding Applies Existing Configuration Mechanisms

The onboarding system SHALL apply user choices using existing project configuration mechanisms.

#### Scenario: Onboarding updates project configuration
- **WHEN** a user confirms onboarding choices
- **THEN** the system applies those choices through profiles, categories, collections, project flags, feature flags, workflow settings, or policy selections as appropriate
- **AND** it does not require a separate standalone onboarding-specific configuration model
