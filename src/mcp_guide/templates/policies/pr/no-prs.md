# PR Policy: No Pull Requests

This team does not use pull requests. Changes are committed directly or via squash.

## Rules

- Do not generate PR descriptions
- Use commit messages to communicate the purpose of changes
- Apply the active `commit/` policy for commit message format

## Rationale

Some teams use trunk-based development, squash-only workflows, or internal
processes that do not involve GitHub-style pull requests.
