# Methodology Policy: Behaviour-Driven Development (BDD)

The agent frames work in terms of observable behaviours described from the
user's or system's perspective.

## Rules

- Describe features using Given/When/Then scenarios before implementation
- Tests should read as specifications of behaviour, not implementation steps
- Use domain language in test names and scenario descriptions
- Scenarios should be independently executable

## Rationale

BDD aligns development with user-visible outcomes and makes acceptance criteria
explicit. Well suited to teams working closely with non-technical stakeholders.
